Detailed Description

The persistent memory driver uses the access driver to really access the persistent memory.

Because this is usually device specific, the persistent memory access has to be implemented by the user.

Function Documentation

void vscp_ps_access_init ( void  )

This function initializes the persistent memory access driver.

It doesn't write anything in the persistent memory! It only initializes the module that read/write access is possible.
